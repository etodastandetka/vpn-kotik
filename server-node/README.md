# VPN-узлы (отдельные сервера)

Папка для **быстрой настройки нового VPS** под узел с панелью (по умолчанию **3x-ui + Xray**). Бот и API остаются в основном репозитории; сюда — только то, что крутится **на каждом сервере с VPN**.

## Когда использовать

- Подняли новый сервер (Ubuntu 22.04/24.04 или Debian 11/12).
- Нужно: базовый файрвол, открытые порты, установка панели, дальше — руками inbound и клиенты.

## Быстрый старт

На **чистом Linux-сервере** (не на Windows):

```bash
# Скопируйте папку server-node на сервер (scp/git) и перейдите в неё
sudo cp env.example .env
sudo nano .env   # порты, при необходимости

chmod +x bootstrap.sh install-3xui.sh
sudo ./bootstrap.sh
```

Установщик 3x-ui обычно **спрашивает пароль админки** — введите в терминале. Если запускали без TTY (редко), см. сообщение в конце скрипта и выполните установку вручную.

## Что делает `bootstrap.sh`

- `apt` — базовые утилиты, **UFW**.
- **UFW**: политика deny/allow, порты из `OPEN_TCP_PORTS` (по умолчанию 22, 80, 443, 2053).
- По `INSTALL_3XUI=true` — скачивает и запускает **официальный** скрипт установки 3x-ui (репозиторий [mhsanaei/3x-ui](https://github.com/mhsanaei/3x-ui)).

Порты панели и протоколов (Reality и т.д.) вы потом задаёте **в панели**; при необходимости добавьте их в `OPEN_TCP_PORTS` и снова откройте в UFW или отдельным `ufw allow`.

## Много серверов

На **каждый** новый VPS:

1. Скопировать эту папку (или весь репозиторий).
2. Свой `.env` (или один шаблон + правка IP/портов).
3. `sudo ./bootstrap.sh`
4. В панели — отдельный inbound/подписка под этот узел.
5. В вашем backend/боте — учёт **нескольких нод** (URL API панели, токены) — это уже код приложения, не эта папка.

## Привязка к боту

Сейчас скрипты **не** подключаются к Telegram-боту автоматически: они только готовят сервер и панель. Дальше вы:

- забираете **subscription URL** / UUID из панели;
- прописываете выдачу в своём API (как задумано в `README` у корня проекта: FastAPI ↔ панель).

**Минимальный чеклист после установки панели**

1. Войти в веб-интерфейс 3x-ui, создать inbound (протокол по вашему выбору).
2. В разделе подписки / клиентов скопировать **Subscription URL** (или экспорт ссылок).
3. На сервере с ботом: в `.env` задать `SUBSCRIPTION_STUB_TEMPLATE` этим URL или шаблоном с `{telegram_id}` / `{plan_id}`, если панель так отдаёт пер-юзер ссылки.
4. Перезапустить контейнеры `api` и `bot`. Для уже выданных подписок при необходимости обновить `subscriptions.config_stub` в БД (см. `scripts/sql/fix_config_stub_by_telegram.sql.example` в корне репозитория).

## Заметки по доступности сервисов из РФ

Доступ к отдельным сайтам и приложениям зависит от **маршрута, DNS, блокировок и политики сервера**. Используйте только **законные** сценарии и правила хостинга. Технически обычно настраивают **полный туннель** (global/VPN) и корректный **DNS** на клиенте; детали — в документации выбранного клиента (Hiddify, v2rayNG и т.д.).

## Безопасность

- Сразу смените пароль панели, не держите панель на `0.0.0.0` без ограничений без необходимости.
- Ограничьте SSH по ключу и при возможности по IP.
- Регулярно обновляйте панель и систему.
